# Gödel and Limitations Policy

Circle Calculus must not claim to escape Gödel's incompleteness theorems.

## Required statement

Any effective, consistent formal system strong enough to represent arithmetic is subject to incompleteness. Circle Calculus can encode syntax, proofs, and glyphs through Gödel-coil numbering, but that encoding is an arithmetization strategy, not an escape hatch.

## Success target

Circle Calculus aims to be:

- a rigorous alternative presentation of mathematics,
- a proof-carrying visual and cyclic syntax,
- a translation layer to standard foundations,
- a system with native advantages for recurrence, periodicity, modularity, provenance, and glyph-based proof search.

It does not aim to be:

- a complete proof system for all mathematical truth,
- a contradiction-proof system by assertion,
- a replacement for rigor,
- a visual proof language without machine-checkable syntax.

## Required language in capstone papers

Use:

```text
Circle Calculus provides a machine-checkable alternative foundation/encoding layer for a large body of mathematics, with explicit translations to standard foundations.
```

Do not use:

```text
Circle Calculus proves every true mathematical statement.
```
