# Circle Calculus Compute 2: CoilRay and CoilSampler

Status: polished draft with a proved direction-bin schedule seed.

## Aim

This paper preserves the rendering and sampling track from the compute handoffs: `CoilRay Sort`, `CoilSampler`, `CoilNoise`, `CoilSTIR`, and angular lighting/sampling compression. These ideas are promising because ray directions, BRDFs, environment maps, procedural samples, and camera/lens samples naturally live on circles, hemispheres, spheres, or phase schedules.

The current formal seed is `COMMON-0020`, the finite direction-bin schedule

```text
bin(bin_count,sample) = sample mod bin_count
```

for positive bin counts. This is a queue/sampler indexing primitive, not a claim that ray binning improves rendering performance.

## Theorem Spine

- `COMPR-T0001`: `Circle.Applications.directionBin_lt_binCount`
- `COMPR-T0002`: `Circle.Applications.directionBin_add_binCount`
- `COMPR-T0003`: `Circle.Applications.directionBin_zero`

## Proved Core

`COMPR-T0001` proves that the bin schedule is bounded by the positive bin count. `COMPR-T0002` proves that adding one full bin count preserves the schedule, and `COMPR-T0003` proves the zero anchor. The Python sidecar checks the same finite examples.

These theorems justify only the finite binning primitive. They do not prove a sphere-binning algorithm, ray coherence, low discrepancy, sampling quality, or rendering speed.

## Rendering Benchmark

The first CoilRay experiment should compare:

```text
baseline wavefront path tracer
direction-bucketed path tracer
spherical-coil-bucketed path tracer
```

At each bounce, map `ray.direction` to an `S^2` bin, map that bin to a coil index, bucket the ray queue, and measure rays per second, divergence, cache behavior, BVH traversal time, noise per sample, and total frame time.

The first CoilSampler experiment should compare deterministic sphere/hemisphere schedules such as golden-angle, spherical Fibonacci, coprime ring schedules, and ordinary random or stratified baselines.

## Guardrail

This is a rendering benchmark track, not a theorem that coil sorting or coil sampling is always faster or better. If benchmarks fail, record the failure.
